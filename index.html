Welcome to your documentation hub

Use this GitBook-inspired theme to centralise the knowledge around your project or product. The layout is optimised for readability with a persistent navigation sidebar, subtle typography, and plenty of room for long-form guides.

Replace the sample content with your own documentation pages. Each section is linked in the sidebar, making it easy for readers to jump directly to what they need.

Tip

You can add more pages by duplicating this template and updating the sidebar links to match your new sections.

Quick start

  1. Plan your documentation structure and define your main sections.
  2. Create a new section in the sidebar for each major topic.
  3. Write focused pages using clear headings and consistent tone.
  4. Publish changes by committing to your GitHub Pages repository.

The styles.css file controls the appearance of the theme. Tweak the color palette or typography to align with your brand.

Project structure

This starter comes with just three files: index.html, styles.css, and script.js. They are small and easy to customise.

File Description
index.html Main layout and content for the documentation homepage.
styles.css Theme styling, including colors, typography, and responsive layout.
script.js Enhancements such as the mobile navigation toggle.

Writing documentation

Keep your guides concise and approachable. Use short paragraphs, descriptive headings, and callouts for important information. This template supports blockquotes, lists, code samples, and tables, so feel free to mix and match content blocks as needed.

Well-crafted docs turn knowledge into a renewable resource for your team. 
// Example: Highlighting a shell command
npm install
npm run docs

Customisation

Adjust the colour palette by editing the CSS variables at the top of the stylesheet. You can also integrate a static site generator or markdown processor if your documentation grows beyond a single page.

  • Update brand colors and fonts
  • Swap the logo text for an SVG asset
  • Connect the search input to your preferred search provider

Deployment

Host your documentation with GitHub Pages, Netlify, Vercel, or any static hosting provider. Whenever you push changes to the main branch, the latest version of your docs will go live.

Frequently asked questions

Can I add more pages?

Absolutely. Duplicate this file and update the navigation. For larger docs, consider a build step to generate pages from Markdown.

Does this support dark mode?

A dark theme is easy to add. Extend the CSS with a [data-theme="dark"] selector and toggle it via JavaScript.

Can I embed rich media?

Yes, you can embed images, videos, or interactive demos just like any HTML page.

API references

Ship interactive API documentation by wiring this site to the Scalar open-source API docs. Scalar renders OpenAPI and GraphQL definitions with an explorer, request builder, and generated snippets so your consumers can understand and try endpoints without leaving the page.

Live request runner

Connect Scalar to your environments to let readers trigger REST or GraphQL calls directly from the documentation, complete with authentication helpers and environment switching.

Schema aware rendering

Scalar automatically expands your OpenAPI models, query parameters, and response schemas with clear typing, markdown descriptions, and example payloads.

Embeddable components

Drop the Scalar viewer into this layout with a lightweight script tag—no separate build step is required and the docs stay synced to your spec repository.

Embed your OpenAPI schema

Paste your schema URL into Scalar’s loader to generate the reference instantly. The snippet below mounts the docs inside a <div id="scalar"> container on this page. 

<link rel="stylesheet" href="https://cdn.scalar.com/static/css/scalar.css" />
<div id="scalar"></div>
<script src="https://cdn.scalar.com/static/js/scalar.min.js"></script>
<script>
  Scalar("scalar", {
    spec: {
      url: "https://example.com/openapi.yaml",
    },
    layout: "modern",
  });
</script>
Explore Scalar open source

Keep your API reference evergreen by sourcing the spec from the same repository as your backend and let Scalar handle the UI, dark mode, and code samples.
